Stream a Live event to the Beacon App

In this topic you will learn how to stream a Live event to the Beacon App.

Create a Live event in Beacon:

In the Beacon platform, click on the Events tab to start the set-up of an Event:

Only activated accounts with "Beacon Live" will have the Playlist and Events available in the tabs section. Please contact Brightcove Support in case you don't have them activated.
  1. Click on Add a new Event
  2. Give the Live event a Name, and an Event Start Date
  3. In the Textual Data tab, fill your event information:
  4. In the Non-Textual Data tab, pick an “Image Poster” for your Event:

    You can select the type of poster you would like for your event between Portrait, Landscape, Thumbnail, and Wide banner.

    Images should be a certain seize
  5. In the Rights & Scheduling tab, you will be able to set the Availability details as:

    Rights Type: Choose the type of rights between Advertising, Subscription or Free.

    Start Time: The start time of your event

    End Time: The end time of your event, these times are created to generate limits within the App.

    Max Streams: Is the max number of streams allowed in the App. The default number is 0 (zero).

    evices Details:D Here you can add the different number of devices you want your stream to be published on. (Example: iOS, Android, Roku, Samsung etc)

    Geo-Location Details: List of countries where you want your app to be enabled. You can choose “World” in case you don’t want to restrict any country.

    Denied Locations: List of regions with denied access to your content.

  6. Click in the button below: Create a new event

Add the Beacon event to a Playlist

Once we create an event, go to the Playlist tab at the upper menu. We need to add the event to a Playlist before we can stream something.

  1. Click on the “Add new playlist” button and create one.
    You need to create a playlist in Beacon that will contain your Live events. Do not use a playlist from Video Cloud, using a Video Cloud playlist may overwrite and delete your Beacon Live recordings.
  2. In Playlist, from the content type tabs, select the Events tab:
  3. Look for your event in the contents module and click on the (+) button to add it to the left playlist. You can position the order of the events in the way you want them to be displayed in the App.
  4. Adjust the settings of your playlist (time, devices, location, etc ...) As you did with your event.
  5. Click on the “Save changes in Playlist” button.
  6. Click on the tool button in the upper right of the screen to clear the Cache, so everything is displayed correctly in the App.

At this point, your event will be created as a container in the Beacon App, but to actually start streaming a Live video we will need to set up our transmission in the Video Cloud Live Platform.

Create a Live event in Studio

To access the Live module, login to Video Cloud Studio and click Live in the navigation header.

  1. From the Manage Events page, click Create Event. The Create Live Event or Create Live Channel page will open. The properties when creating events and channels are the same. The screenshot in this document covers creating an event.
  2. Enter the event details:
    • Event Name - Name for the event; this name will also appear in the Media module if the video is saved as a video asset (max 255 characters)
    • Short Description - Description of the event (max 250 characters)
    • Tags - Tags to associate with the event; tags will be assigned to the video if the video is saved as a video asset (max 128 characters and 1200 tags)
    • Reference ID - Reference ID for the event; must be unique (max 150 characters)
  3. Any required custom fields for the account will be displayed. Assign values to the custom fields.
  4. Setting Streaming to Social Media to Allowed will make the event available for streaming to social media platforms. This can be done once the event has started streaming or by using the Social module.
  5. Select a Live Ingest Profile. The dropdown list will list all Live ingest profiles. The ingest profile will control the quality and number of live renditions that are created. Clicking the Manage Profiles link will open the ingest profile editor in another browser tab. Note that you must be an account administrator to access the ingest profile editor. Click the RENDITIONS link to view the renditions that will be created using the selected ingest profile.
    The bitrate and billing details of the three tiers are as follows:
    • Tier 1 allows up to 5,500 Kbps and is charged at 1.0x hours
    • Tier 2 allows up to 10,000 Kbps and is charged at 1.5x hours
    • Tier 3 allows up to 20,000 Kbps and is charged at 2.0x hours
  6. Select Convert event to video asset when complete to save a VOD version of the live video to your Video Cloud account. When this option is selected, the highest quality rendition from the selected Live Ingest Profile is used as the source for the VOD video. The default ingest profile for the account is used when creating renditions. Note that the VOD version is not available as soon as the live event ends, the video will be available once the ingestion process is complete.
    Note: This option is not supported when using a recurring event.
  7. Click ADVANCED OPTIONS and configure the advanced options.
    Note: For Beacon Live work, you have to enable Stream Status Notifications checkbox in the Advanced options.
  8. Enable stream status notifications

    Add the notifications URL from Beacon CMS (this URL will be provided by Brightcove Support)

  9. Click Create Event to start the live event. The Control Room page will open with details about the live event including the Streaming Endpoint (RTMP URL) and Stream Name which will be used by your encoder.

    It will take about a minute to process your streams.

    To start streaming in the Live platform, you will need an Encoder, for information on how to use an encoder please read the following the documentation:

    Step-by-Step: Using the Live Module with Telestream Wirecast

    Step-by-Step: Using the Live Module with Open Broadcaster Software (OBS)

  10. In the Control Room copy the “Job Id” and the “Video ID” from the Test Event panel, we will need both to configure the Live event in the Beacon platform settings.
  11. Next, go back to Beacon and click in the Event tab, go to the event we previously created there, and in the Basic Data settings paste the Job ID of your Live event.

    Then go to the Streams tab and create a new Stream, in the Stream Video ID, paste the Video ID from the Video Cloud Event we created before.

    Fill in the following information:

    • Select the number of devices in which you can publish your video.
    • Change the Stream Type to: Live
    • And click the “Update event” button before leaving the page:
  12. Clear your cache.

    Go to the Beacon App and Select your playlist and event. Click on Play.

    You should be able now to see your Live streaming from Video Cloud in the Beacon App.

    Once your event ends, you can save it as a clip to transmit it later as a VOD on Beacon.

    13. Create a Clip from your Beacon Live event

    You have to create a clip of your Live Beacon event if you want to keep the video for streaming later on as a VOD recorded video.

    The Live module provides the ability to create video clips from completed events. Clips can also be created while a live event is in progress.

    For a complete Clipping feature list, please review the Creating a Video Clip Using the Live Module documentation.

    1. In the Live module go to the “Create Clip” tab in the left panel
    2. Using the clipping buttons or shortcut keys, set the start and endpoints for the clip. Or just click on the Save button to keep the entire video.
    3. Enter the clip video properties.
      • Clip Name - Name of the clip
      • Short Description - Description of the clip
      • Tags - Tags to associate with the clip
      • Reference ID - Reference ID to assign to the clip
      • Remove Source Ad Break Content - This option removes ad breaks triggered within the clip, keeping only program content. Note: The Remove Source Ad Break checkbox will only appear when the stream is SSAI enabled.
    4. Click Create Clip. The transcoding process for the clip will begin. Clips are transcoded using the default ingest profile for the account.

    Add your Clip to Beacon as a playable stream video.

    1. Take the number ID of your created clip:
    2. In the Beacon platform, go to the Events tab and select the Streams menu, change the Stream Type in the dropdown from Live to Stream, and in the Stream Video ID option, paste the ID number you just copied from your Clip event in the Live platform.
    3. Click in the Basic Data tab and change the Status from Ended to Finished. Before leaving click the Update Event button at the bottom of the page, this will make the Play button available in the App to see the recording of the event.

    (Static Entry Point) SEP Live events

    SEP are recurring events that you can let activated/deactivated when your event starts over and over again as for a weekly program, for example. This event can be used as a one for all Live Job ID that can be transmitted to several channels in the Beacon App.

    • Note that when you create a recurring event, a video is automatically created for it in the Media module. Do not delete this video, as it will be needed every time the event recurs.
    • The recurring event should be activated before the encoder is started.

    To create a recurring event, select the Create this as a recurring event using a Static Entry Point (SEP) option when creating a live event and then click Create Event.

    1. In the Live Platform create a new Event. For more information in how to create an event please read the followingdocumentation.
    2. Expand the Advanced Options, and enable the “Create this as a recurring event using a Static Entry Point (SEP)” option. For more information in SEP clickhere.
    3. When you already have a SEP event, click on the Standby tab, and you will be able to activate/deactivate it from the Live Control Room.
    4. Copy your Live Job ID
    5. Return to the Beacon platform and configure your events giving them a name and a scheduled time.
    6. The date and time of your event must be in the following format:
    7. Click in the Update event button.

    You can perform this process with all the event channels you want in the Beacon platform, and all of them will be streaming from the same Live event transmission, but each one with the different date and time that you adjust in each of its event settings.
